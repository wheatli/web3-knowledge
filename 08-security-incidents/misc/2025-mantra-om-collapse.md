# MANTRA (OM) 代币 24 小时闪崩 90%+

> **TL;DR**：2025-04-13 周日深夜 UTC，RWA（现实世界资产）代币 MANTRA (OM) 在 90 分钟内由约 $6.3 下跌至 $0.50，随后一度触及 $0.37，市值从约 $60 亿蒸发至 $5 亿以下。事件非链上漏洞，而是疑似大额持仓人（或做市商 / 内部人）在 CEX 流动性薄弱时段的集中抛售触发级联强平，叠加代币流通量 & 解锁安排公开性不足引发的信任危机。MANTRA 团队与 CEO John Mullin 否认内部抛售，将事件归因"CEX 强制平仓"，但社区与 ZachXBT 指出多个链上地址向交易所的集中流入早于下跌，争议未定。事件被行业记入"2025 重大信任 / 市场结构事件"。

## 1. 事件背景

- **MANTRA Chain / OM 代币**：主打 RWA (Real World Asset) 代币化，基于 Cosmos SDK 的应用链，OM 为其治理与质押代币。2024 下半年至 2025-Q1 OM 价格从 $0.10 上涨至约 $9，涨幅 90x+，成为 2024-2025 RWA 叙事最大赢家之一。
- **事件前**：FDV 超 $60 亿，流通量约 $10 亿级（具体因团队/基金会持仓未完全 on-chain 透明而有争议）。
- **时间轴**（UTC）：
  - 2025-04-13 18:00 左右：链上监控发现多个疑似基金会 / 早期投资人地址向 CEX 集中转账数千万枚 OM。
  - ~20:00–21:30：Binance / OKX 等 CEX 上 OM 出现大额卖盘，价格从 $6.3 跌破 $3，触发 CEX 永续合约的级联清算。
  - 21:30–22:00：进一步跌至 $0.50 以下，盘中最低约 $0.37。
  - 2025-04-14 早间：MANTRA CEO John Mullin 发声明，否认团队抛售，将原因归咎"某 CEX 在周日低流动性时段进行无预警强平"。
  - 随后几天 OM 价格部分反弹到 $0.90–1.20 区间，但叙事已重伤。

### 发现过程

- ZachXBT、Arkham、Lookonchain 等链上侦探同步报告"抛售前数天有大户向 CEX 集中转入"。
- Chainalysis 后续发布分析报告，关联多个地址簇。

## 2. 事件影响

### 直接"损失"（市值）

- OM 市值短时蒸发约 $50–55 亿（峰值 FDV 对比低点）。不是资金"被盗"，而是持有者账面财富瞬间折损。
- CEX 永续合约爆仓盘据 Coinglass 数据单日约 $60–80M（多空双爆）。

### 受害方

- 二级市场长线持有 OM 的散户与基金。
- 在 CEX 使用 OM 做保证金或质押借贷的用户被连环清算。

### 连带影响

- RWA 赛道整体回调 15–25%（ONDO、POLYX、SNSY 等）。
- 行业重提"代币流通量披露透明度"标准（CoinGecko、CMC 流通量的可靠性争议）。
- CEX 上币标准、OTC 大户代币入金监控流程受关注。

### 资金去向

- 不涉及"资金被盗"，但 ZachXBT 指出至少 17 个钱包在抛售前向 OKX 转入约 43.6M OM（~$230M 名义值），归属未定。
- 若最终证实由内部人 / 早期投资人预先抛售，将构成严重的信任 / 合规问题。MANTRA 团队对此否认。

## 3. 技术 / 结构根因（非代码漏洞）

### 3.1 类别

**Governance / Tokenomics / Market-Structure / Trust** — 不是合约漏洞，不是黑客攻击，而是"流通量信息不对称 + 低流动性时段大户集中抛售 + CEX 强平级联"的综合。

### 3.2 机理

1. **流通量披露不对称**：市场公开流通量与实际可抛售量差距较大；若某些锁仓代币已私下场外流通或通过做市协议借出，报价数据失真。
2. **杠杆堆叠**：OM 在 2024-Q4 - 2025-Q1 的上涨过程中，CEX 永续未平仓合约持续走高；大量散户以 OM 本身做保证金 / 用高杠杆多头头寸。
3. **低流动性时段**：周日 UTC 夜间是全球市场流动性谷底，抛售 + 清算传递放大。
4. **连锁清算**：现货暴跌 → perp mark price 跌 → 杠杆多头被强平 → 市价卖单吃掉订单簿 → 价格再跌，循环 15 分钟即可完成 90% 跌幅。

### 3.3 归因之争

- **团队口径**：某 CEX 主动强平某"不合规"借贷头寸，未提前通知，导致抛压传导。
- **社区 / ZachXBT 口径**：多个地址在下跌前数天已将大量 OM 流入 CEX，且这些地址与团队 / 早期投资人存在可疑关联。
- 截至 last_verified，无执法机构正式定性；分类为"未有结论的市场操纵 / 内部人抛售争议"。

## 4. 事后响应

- MANTRA 团队：多次公开声明；公布"销毁团队代币 1.5 亿 OM" 的提议回应信任危机；加强代币解锁透明披露。
- 主流 CEX 未公开承认"主动强平"行为；部分交易所调整 OM 的杠杆 / 保证金参数。
- 链上侦探与合规平台（Chainalysis、Arkham、Nansen）发布多篇事后分析。
- 未见 FBI / SEC / 其他监管正式调查的公开披露，状态"未公开"。

## 5. 启发与教训

- **流通量 & 解锁披露标准化**：行业需要统一 + 可验证（链上 merkle / multisig timelock）的代币流通与解锁披露；CoinGecko/CMC 单方面数据不足。
- **代币经济学的 "黑色星期日" 压力测试**：项目方应定期对"周日夜间 + 30% 持仓抛售"情景做流动性压力测试。
- **CEX 风控**：对单币种持仓集中度设硬上限；大户入金触发自动风控审查。
- **用户**：警惕"纯叙事驱动 + 流通量不透明"的高位代币；不要用高 beta 代币做杠杆保证金。
- **与"黑客"事件并列的价值**：2025 年 OM 事件说明"安全"不仅是代码漏洞——治理、信息披露、市场结构同样是影响用户资产的"攻击面"，应纳入安全知识库。
